# 03-03: Zustand State Management Setup

## Requirements
- MGMT-01: 用户可按用途分类管理提示词模板
- DATA-01: 提示词数据本地持久化存储

## Summary
Create a Zustand store for managing prompts and categories state with CRUD operations, category filtering, and storage synchronization. This provides the reactive state layer for the Popup UI.

## Tasks

### Task 1: Create Zustand Store
<read_first>
- src/shared/types.ts: Prompt and Category interfaces
- src/shared/messages.ts: MessageType.GET_STORAGE and SET_STORAGE for storage sync
- src/lib/storage.ts: StorageManager from Plan 03-01 for default data reference
</read_first>

<action>
Create `src/lib/store.ts`:

1. Install Zustand (if not already installed):
   - Run: `npm install zustand`

2. Create store with:
   ```typescript
   import { create } from 'zustand'
   import type { Prompt, Category, StorageSchema } from '../shared/types'
   import { MessageType } from '../shared/messages'

   interface PromptStore {
     prompts: Prompt[]
     categories: Category[]
     selectedCategoryId: string | null
     isLoading: boolean
     
     // Actions
     loadFromStorage: () => Promise<void>
     saveToStorage: () => Promise<void>
     setSelectedCategory: (categoryId: string | null) => void
     
     // Prompt CRUD
     addPrompt: (prompt: Omit<Prompt, 'id'>) => void
     updatePrompt: (id: string, updates: Partial<Prompt>) => void
     deletePrompt: (id: string) => void
     
     // Category CRUD
     addCategory: (name: string) => void
     deleteCategory: (id: string) => void
     
     // Computed getters
     getPromptsByCategory: (categoryId: string) => Prompt[]
     getFilteredPrompts: () => Prompt[]
   }
   ```

3. Implement store with:
   - `loadFromStorage`: Uses chrome.runtime.sendMessage with GET_STORAGE, sets prompts/categories
   - `saveToStorage`: Uses chrome.runtime.sendMessage with SET_STORAGE with current state
   - `addPrompt`: Generates UUID id, adds to prompts array, calls saveToStorage
   - `updatePrompt`: Updates prompt by id in prompts array, calls saveToStorage
   - `deletePrompt`: Removes prompt by id from prompts array, calls saveToStorage
   - `addCategory`: Generates UUID id, sets order to categories.length + 1, calls saveToStorage
   - `deleteCategory`: Removes category, moves its prompts to 'default' category, calls saveToStorage
   - `getPromptsByCategory`: Returns prompts filtered by categoryId
   - `getFilteredPrompts`: Returns prompts for selectedCategoryId or all if null

4. Generate IDs using:
   - Import: `import { v4 as uuidv4 } from 'uuid'`
   - Or use crypto.randomUUID() if uuid package not installed

5. Initialize with empty state:
   - prompts: []
   - categories: [{ id: 'default', name: '默认分类', order: 0 }]
   - selectedCategoryId: 'default'
   - isLoading: true
</action>
