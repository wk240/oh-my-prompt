# Plan 08: Build & Extension Load Verification

**Objective:** Build the extension and verify it loads successfully in Chrome with all Phase 1 success criteria.

---

## Task 01-08-01: Install dependencies

<read_first>
- package.json (created in Plan 01)
</read_first>

<action>
Run `npm install` in project root directory.

Expected outcome: node_modules/ directory created, all dependencies installed without errors.
</action>

<acceptance_criteria>
- node_modules/ directory exists
- @crxjs/vite-plugin installed (check node_modules/@crxjs)
- react and react-dom installed
- vite and typescript installed
- No install errors in console output
</acceptance_criteria>

---

## Task 01-08-02: Build extension

<read_first>
- All Phase 1 source files created
- vite.config.ts (Plan 01)
- manifest.json (Plan 02)
</read_first>

<action>
Run `npm run build` (or `npm run dev` for development build).

Expected outcome: dist/ directory created with bundled extension files.
</action>

<acceptance_criteria>
- dist/ directory exists
- dist/ contains manifest.json
- dist/ contains bundled JS files for background, content, popup
- dist/ contains assets/ with icons
- No build errors in console output
</acceptance_criteria>

---

## Task 01-08-03: Load extension in Chrome

<read_first>
- dist/ build output
- manifest.json in dist/
</read_first>

<action>
Manual verification steps:

1. Open Chrome browser
2. Navigate to chrome://extensions
3. Enable "Developer mode" toggle (top right)
4. Click "Load unpacked" button
5. Select the dist/ folder from file dialog
6. Verify extension appears in list without errors
</action>

<acceptance_criteria>
- Extension appears in chrome://extensions list
- Extension name shows "Lovart Prompt Injector"
- No error messages displayed
- Extension icon appears in Chrome toolbar
</acceptance_criteria>

---

## Task 01-08-04: Verify Service Worker

<read_first>
- Service Worker loaded
</read_first>

<action>
Manual verification:

1. In chrome://extensions, click "Service Worker" link under extension details
2. DevTools opens for Service Worker
3. Check Console for: `[Lovart Injector] Service Worker started`
4. In Console, run test: `chrome.runtime.sendMessage({type: 'PING'}, console.log)`
5. Verify response: `{success: true, data: 'pong'}`
</action>

<acceptance_criteria>
- Service Worker DevTools Console shows startup log
- PING message returns `{success: true, data: 'pong'}` response
</acceptance_criteria>

---

## Task 01-08-05: Verify Content Script injection

<read_first>
- Content Script loaded
- Lovart domain accessible (or mock)
</read_first>

<action>
Manual verification:

1. Navigate to lovart.ai (if accessible)
2. Open DevTools Console (F12)
3. Verify log: `[Lovart Injector] Content script loaded on: ...`
4. Verify log: `[Lovart Injector] Ping response: {success: true, data: 'pong'}`

If Lovart.ai not accessible:
1. Check manifest.json content_scripts matches `*://*.lovart.ai/*`
2. Extension load verification confirms manifest is valid
</action>

<acceptance_criteria>
- If Lovart accessible: Console shows content script loaded log
- If Lovart not accessible: manifest.json confirmed correct match pattern
</acceptance_criteria>
