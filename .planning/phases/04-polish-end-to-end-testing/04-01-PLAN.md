# Plan 04-01: Error Handling Expansion

## Objective

Expand Toast error notifications to cover all error scenarios across import/export, storage operations, and popup interactions. Ensure users receive clear, actionable error messages without disrupting their workflow.

## Context

Phase 3 established the Toast notification system using shadcn/ui components. Phase 4 extends this coverage to handle additional error scenarios identified in D-04/D-05/D-06 decisions. All error messages use the existing Toast system with `variant: 'destructive'` for consistency.

**Established patterns:**
- Toast hook: `useToast()` from `src/hooks/use-toast.ts`
- Toast display: `{ title: 'Error Title', description: 'Error details', variant: 'destructive' }`
- Console log prefix: `[Lovart Injector]`

## Tasks

### Task 1: Enhance Import Validation Error Messages

**Priority:** high

<read_first>
- src/lib/import-export.ts (current validation error messages)
- src/popup/App.tsx (import flow and Toast usage)
</read_first>

<action>
1. In `src/lib/import-export.ts`, update the `validateImportData` function to return more specific error messages for each validation failure case:

   - Replace `'prompts 中包含无效数据'` with `'提示词数据格式错误：每项需包含 id、name、content、categoryId'`
   - Replace `'categories 中包含无效数据'` with `'分类数据格式错误：每项需包含 id、name、order'`
   - Replace `'提示词结构不完整'` with `'提示词缺少必要字段：id、name、content、categoryId'`

</action>

### Task 2: Add Storage Operation Error Toasts

**Priority:** high

<read_first>
- src/lib/store.ts (current error handling in loadFromStorage and saveToStorage)
- src/popup/App.tsx (loadFromStorage usage in useEffect)
</read_first>

<action>
1. In `src/lib/store.ts`, modify the `loadFromStorage` function to return a boolean indicating success/failure:
   - Change return type from `Promise<void>` to `Promise<{ success: boolean; error?: string }>`
   - On catch error, return `{ success: false, error: '数据加载失败，请检查存储权限' }`
   - On success, return `{ success: true }`

2. Modify `saveToStorage` similarly:
   - Change return type from `Promise<void>` to `Promise<{ success: boolean; error?: string }>`
   - On catch error, return `{ success: false, error: '数据保存失败，请检查存储配额' }`

3. In `src/popup/App.tsx`, update `loadFromStorage` usage:
   - Change `useEffect` to check the result and show Toast on failure
   - Add: `const result = await loadFromStorage()` and check `if (!result.success)` then show error Toast

4. Export a new function `checkStorageQuota` in `src/lib/storage.ts`:
   - Use `chrome.storage.local.getBytesInUse(STORAGE_KEY)` to get current usage
   - Chrome storage.local quota is 10MB (10,485,760 bytes)
   - Return `{ usedBytes: number, quotaBytes: number, percentage: number }`
   - If percentage > 80%, log warning with `[Lovart Injector] Storage usage warning: ${percentage}%`
</action>

<acceptance_criteria>

- grep -c "数据保存失败" src/lib/store.ts returns >= 1
- grep -c "getBytesInUse" src/lib/storage.ts returns >= 1
- grep -c "checkStorageQuota" src/lib/storage.ts returns >= 1
</acceptance_criteria>
