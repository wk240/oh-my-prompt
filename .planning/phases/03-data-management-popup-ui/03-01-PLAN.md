# 03-01: Storage Manager Implementation

## Requirements
- DATA-01: 提示词数据本地持久化存储

## Summary
Implement a Storage Manager module that handles all chrome.storage.local operations with proper TypeScript types, error handling, and default data initialization. This provides the foundation for all data persistence in Phase 3.

## Tasks

### Task 1: Create Storage Manager Module
<read_first>
- src/shared/types.ts: StorageSchema interface defines the data structure
- src/shared/constants.ts: STORAGE_KEY and DEFAULT_CATEGORY_NAME constants
- src/content/sample-data.ts: SAMPLE_PROMPTS and SAMPLE_CATEGORIES for default data template
</read_first>

<action>
Create `src/lib/storage.ts` with the following exports:

1. `StorageManager` class with methods:
   - `getData(): Promise<StorageSchema>` - Retrieves full storage data
   - `saveData(data: StorageSchema): Promise<void>` - Saves full storage data
   - `getPrompts(): Promise<Prompt[]>` - Retrieves prompts array
   - `getCategories(): Promise<Category[]>` - Retrieves categories array
   - `getDefaultData(): StorageSchema` - Returns default data structure with:
     - version: '1.0.0'
     - categories: [{ id: 'default', name: '默认分类', order: 0 }]
     - prompts: [] (empty array, user will add their own)

2. Error handling:
   - Wrap all chrome.storage.local calls in try-catch
   - Log errors with prefix `[Lovart Injector]`
   - Return default data on storage empty or error

3. Implementation details:
   - Use `chrome.storage.local.get(STORAGE_KEY)` for retrieval
   - Use `chrome.storage.local.set({ [STORAGE_KEY]: data })` for saving
   - Import types: Prompt, Category, StorageSchema from '../shared/types'
   - Import constants: STORAGE_KEY, DEFAULT_CATEGORY_NAME from '../shared/constants'
</action>
