# Plan 04-06: Extension Packaging

## Objective

Prepare the Chrome extension for release packaging. Verify version numbers are synced to 1.0.0, execute a clean production build, and document the build/package process for future releases.

## Context

Phase 4 completes the extension polish. This plan ensures the extension is ready for distribution:
- Version numbers must be synced between manifest.json and package.json at 1.0.0 for first release
- Production build must generate a complete, loadable extension bundle
- Build process documentation enables future releases and developer handoff

**Build Configuration:**
- `vite.config.ts` uses @crxjs/vite-plugin for CRX bundling
- Output directory: `dist/`
- Source maps enabled for debugging (sourcemap: true)
- Base path: `./` (relative paths for extension)

**Current State:**
- manifest.json version: "1.0.0" (already correct)
- package.json version: "1.0.0" (already correct)

### Task 4: Document Build Process

**Priority:** medium

<read_first>
- vite.config.ts (build toolchain reference)
- package.json (available npm scripts)
</read_first>

<action>
1. Create or update a BUILD.md file in project root documenting:

   **Build Commands:**
   ```markdown
   ## Build Commands

   - Development: `npm run dev` - Hot reload enabled
   - Production: `npm run build` - Creates dist/ folder
   - Preview: `npm run preview` - Preview production build locally

   ## Build Output
   The production build outputs to `dist/` directory:
   - `manifest.json` - Extension manifest
   - `src/` - Compiled content scripts and background
   - `assets/` - Static assets including icons
   ```

   **Chrome Loading Instructions:**
   ```markdown
   ## Loading in Chrome
   1. Open chrome://extensions/
   2. Enable "Developer mode"
   3. Click "Load unpacked"
   4. Select the `dist/` directory

   ## Creating .crx Package
   In chrome://extensions/ with Developer mode:
   1. Click "Pack extension" button
   2. Extension root directory: `dist/`
   3. Private key: Leave empty for first pack (key will be generated)
   4. Output: `.crx` file in project root

   Note: Keep the generated `.pem` private key secure for future updates.
   ```

2. Document version bump process for future releases:
   ```markdown
   ## Version Update Process
   1. Update version in manifest.json
   2. Update version in package.json (sync with manifest)
   3. Run `npm run build` to regenerate dist/
   4. Load in Chrome for smoke test
   5. Pack extension if ready for release
   ```

3. Document known build considerations:
   - Source maps are enabled (sourcemap: true) for debugging
   - TypeScript strict mode - ensure no compilation errors
   - @crxjs/vite-plugin handles manifest transformation automatically
</action>

- grep -c "Load unpacked" BUILD.md returns >= 1
- grep -c ".crx" BUILD.md returns >= 1
- grep -c "Version Update Process" BUILD.md returns >= 1
</acceptance_criteria>
