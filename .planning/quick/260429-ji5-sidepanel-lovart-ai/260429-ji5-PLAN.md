# Quick Task Plan: Sidepanel Universal Input Detection

## Goal

让提示词插入功能在任意页面可用，只要检测到页面有输入框，就可以插入，不再限制只在 lovart.ai。

## Context

当前架构：
- **lovart.ai** → content-script.ts (完整功能：下拉菜单 + 输入检测 + 插入)
- **其他页面** → vision-only-script.ts (仅 Vision Modal，无插入功能)

问题：SidePanelApp 检测 `isOnLovart` 仅基于 URL，导致其他页面的输入框无法使用插入功能。

## Tasks

### Task 1: 添加 CHECK_INPUT_AVAILABILITY 消息类型

**Files:** `src/shared/messages.ts`

**Action:**
- 添加 `CHECK_INPUT_AVAILABILITY` 消息类型，用于 sidepanel 查询当前页面是否有可用输入框

**Verify:**
- MessageType 枚举包含新类型

**Done:**
- `grep "CHECK_INPUT_AVAILABILITY" src/shared/messages.ts` 有输出

---

### Task 2: 增强 vision-only-script 支持通用输入检测和插入

**Files:** `src/content/vision-only-script.ts`

**Action:**
1. 引入 InputDetector 类或实现简化版通用输入检测
2. 添加 MutationObserver 检测通用输入框（textarea, input[type=text], contenteditable）
3. 处理 `INSERT_PROMPT_TO_CS` 消息，找到当前输入框并插入
4. 处理 `CHECK_INPUT_AVAILABILITY` 消息，返回是否有可用输入框
5. 使用 InsertHandler 或简化版 execCommand 方式插入

通用输入选择器：
```typescript
const UNIVERSAL_INPUT_SELECTORS = [
  'textarea:not([readonly])',
  'input[type="text"]:not([readonly])',
  'input[type="search"]:not([readonly])',
  'input:not([type]):not([readonly])',  // 默认 text
  'div[contenteditable="true"][role="textbox"]',
  'div[contenteditable="true"]',
]
```

**Verify:**
- vision-only-script.ts 包含 inputElement 状态跟踪
- 包含 INSERT_PROMPT_TO_CS 和 CHECK_INPUT_AVAILABILITY 处理逻辑

**Done:**
- `grep "INSERT_PROMPT_TO_CS" src/content/vision-only-script.ts` 有输出
- `grep "CHECK_INPUT_AVAILABILITY" src/content/vision-only-script.ts` 有输出

---

### Task 3: 修改 SidePanelApp 改为检测输入框可用性

**Files:** `src/sidepanel/SidePanelApp.tsx`

**Action:**
1. 将 `isOnLovart` 状态改为 `hasInputAvailable`（或保留两者，`hasInputAvailable` 作为主要判断）
2. 在 useEffect 中发送 `CHECK_INPUT_AVAILABILITY` 消息查询输入框状态
3. 修改 `handleSelectPrompt` 和 `handleInjectResource` 使用 `hasInputAvailable` 判断
4. 更新 UI 提示：从 "当前页面不支持提示词插入" 改为更通用的提示
5. 移除或调整 "前往 Lovart" 按钮（保留为可选入口）

**Verify:**
- sidepanel 不再仅依赖 URL 判断，而是查询实际输入框状态
- 非 lovart 页面如有输入框也能插入

**Done:**
- 在任意页面（如 Google 搜索）打开 sidepanel，检测输入框可用时能插入提示词
