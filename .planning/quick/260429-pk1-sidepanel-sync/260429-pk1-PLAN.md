# Quick Task: Sidepanel Storage Change Detection & Auto-sync

## Problem

侧边栏(SidePanelApp.tsx)仅在mount时调用`loadFromStorage()`一次,没有监听存储变更。
当backup页面导入数据或service worker广播REFRESH_DATA时,侧边栏不会刷新数据。

## Reference Pattern

Content script (content-script.ts) handles data refresh:
1. Line 63-74: Handles `MessageType.REFRESH_DATA` → calls `loadFromStorage()`
2. Service worker broadcasts REFRESH_DATA to all tabs after backup restore

## Solution

Add two mechanisms to SidePanelApp.tsx:

### Task 1: Add chrome.runtime.onMessage listener

Add message listener for REFRESH_DATA to refresh data when backup page triggers it.

**Files:** `src/sidepanel/SidePanelApp.tsx`
**Action:** Add useEffect with chrome.runtime.onMessage listener
**Verify:** Console shows "[Oh My Prompt] SidePanel: Data refreshed" when backup import
**Done:** Listener properly cleaned up on unmount

### Task 2: Add chrome.storage.onChanged listener

Add storage change listener for more robust detection (catches any storage change).

**Files:** `src/sidepanel/SidePanelApp.tsx`
**Action:** Add useEffect with chrome.storage.onChanged listener
**Verify:** Sidepanel updates immediately when storage changes
**Done:** Listener properly cleaned up on unmount

## Implementation Details

```typescript
// Add to SidePanelApp.tsx after line 754 (after loadFromStorage useEffect)

// Listen for REFRESH_DATA messages from service worker
useEffect(() => {
  const handleMessage = (message: { type: string }) => {
    if (message.type === MessageType.REFRESH_DATA) {
      console.log('[Oh My Prompt] SidePanel: Received REFRESH_DATA, refreshing...')
      loadFromStorage()
    }
  }
  chrome.runtime.onMessage.addListener(handleMessage)
  return () => {
    chrome.runtime.onMessage.removeListener(handleMessage)
  }
}, [loadFromStorage])

// Listen for storage changes directly (robust fallback)
useEffect(() => {
  const handleStorageChange = (changes: { [key: string]: chrome.storage.StorageChange }) => {
    if (changes[STORAGE_KEY]) {
      console.log('[Oh My Prompt] SidePanel: Storage changed, refreshing...')
      loadFromStorage()
    }
  }
  chrome.storage.onChanged.addListener(handleStorageChange)
  return () => {
    chrome.storage.onChanged.removeListener(handleStorageChange)
  }
}, [loadFromStorage])
```

## must_haves

- STORAGE_KEY import from shared/constants.ts
- MessageType import already exists
- loadFromStorage from usePromptStore already exists
- Cleanup listeners on unmount
