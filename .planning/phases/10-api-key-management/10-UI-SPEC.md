## Interaction Patterns

### Save Button Behavior
1. User clicks "保存配置"
2. Client-side validation runs:
   - Base URL: must start with `http://` or `https://`
   - API Key: non-empty after trim
   - Model Name: non-empty after trim
3. If validation fails: display error message, button remains enabled
4. If validation passes:
   - Button shows loading state: "保存中..." + disabled
   - Send `SET_API_CONFIG` message to service worker
   - On success: display "配置已保存", clear error, reset loading
   - On failure: display "保存配置失败，请重试", reset loading

### Delete Button Behavior
1. User clicks "删除配置"
2. Open Dialog with confirmation:
   - Title: "确认删除"
   - Description: "此操作将删除您的 API 配置，是否继续？"
   - Cancel button (outline variant)
   - Confirm button (destructive variant)
3. User confirms:
   - Confirm button shows "删除中..." + disabled
   - Send `DELETE_API_CONFIG` message
   - On success: close dialog, display "配置已删除", clear inputs
   - On failure: display error, reset loading

### Loading States
- Button text changes to action verb + "中..."
- Button becomes disabled (opacity-50)
- Icons remain visible (do not replace with spinner for simplicity)

### Error/Success Feedback
- Error: `text-sm text-red-500` displayed inline above buttons
- Success: `text-sm text-green-600` displayed inline above buttons
- Auto-clear: Error clears on successful action; Success fades after 3 seconds (optional)

### Input Focus States
- Focus ring: `focus-visible:ring-2 focus-visible:ring-ring`
- Focus ring offset: `focus-visible:ring-offset-2`
- Border color change: `focus:border-gray-400` (enhanced visibility)

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Label association | Each input has `<label>` with matching ID |
| Focus visibility | Tailwind focus-visible ring (2px ring with offset) |
| Keyboard navigation | Tab order: Base URL → API Key → Model Name → Save → Delete → Close |
| Error announcement | Error messages use `role="alert"` or `aria-live="polite"` |
| Button labels | All buttons have text labels (not icon-only) |
| Dialog accessibility | Radix UI Dialog handles focus trap, escape key, screen reader announcements |
| Screen reader hints | Input placeholders serve as hints; additional hint text below buttons |

**ARIA attributes:**
- Inputs: `aria-describedby` pointing to error message when error present
- Dialog: Radix UI provides `aria-modal`, `aria-labelledby`, `aria-describedby`
- Close button: `aria-label="关闭"`
