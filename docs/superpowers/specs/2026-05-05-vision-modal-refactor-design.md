# VisionModal 左右布局重构设计

## 概述

将 VisionModal 和 BatchProgressPanel 合并为统一的左右布局弹窗，提供一致的单图和多图分析体验。

**核心目标：**
- 左侧展示图片预览列表（可切换），点击切换右侧详情
- 右侧展示完整的提示词分析内容
- 统一状态管理和交互体验

## 架构变化

### 删除的文件/组件
- `src/content/components/BatchProgressPanel.tsx` — 移除
- `src/content/core/task-queue-store.ts` — 简化，移除 `isPanelOpen` 状态
- BatchProgressPanel 相关样式代码

### 保留并修改的文件
- `src/content/vision-modal-manager.tsx` — 保持入口，调用重构后的 VisionModal
- `src/content/components/VisionModal.tsx` — 重构为左右布局
- `src/content/core/task-queue-manager.ts` — 保留核心逻辑，移除关闭面板的副作用

### 新的数据流
```
用户点击图片 → TaskQueueManager.addTask()
                    ↓
              useTaskQueueStore.tasks[]
                    ↓
              VisionModal 订阅 store
                    ↓
         左侧列表渲染 + 右侧详情展示
```

### 状态管理简化
- `useTaskQueueStore` 只管理 `tasks` 数组
- VisionModal 自己管理：
  - `isPanelOpen`（是否显示弹窗）
  - `selectedTaskId`（当前选中的任务ID）

## 左侧列表区域设计

### 布局尺寸
- 左侧面板宽度：120px
- 右侧内容宽度：480px（与现有 VisionModal 一致）
- 整体弹窗宽度：600px

### 列表项结构
```
┌─────────────────┐
│  ┌──────────┐   │  ← 缩略图 60x60
│  │  图片    │   │
│  └──────────┘   │
│     ○ 状态      │  ← 状态图标（居中显示）
└─────────────────┘
```

### 四种状态的视觉表现
| 状态 | 图标 | 颜色 |
|------|------|------|
| pending | 灰色圆点 | #9CA3AF |
| running | 旋转 Loader2 | #171717 |
| success | 绿色 Check | #22c55e |
| failed | 红色 X | #ef4444 |

### 交互行为
- 点击列表项 → 切换右侧详情显示该任务的内容
- 当前选中项 → 蓝色边框高亮（`border: 2px solid #3b82f6`）
- Hover 效果 → 背景变为 `#f0f0f0`
- 无图片时 → 显示占位符"无图片"

### 滚动行为
- 左侧列表固定高度（300px），超出时滚动
- 新任务添加后自动滚动到顶部
- 支持鼠标滚轮滚动

## 右侧内容区域设计

### 内容结构（保持与现有 VisionModal 一致）
```
┌──────────────────────────────────────────┐
│  标题（如有）                             │
├──────────────────────────────────────────┤
│  ┌────────────────────────────────────┐  │
│  │ 提示词预览（可滚动，max-height: 200px）│ │
│  │                                    │  │
│  │                      [复制按钮]    │  │
│  └────────────────────────────────────┘  │
├──────────────────────────────────────────┤
│  分析说明（如有）                         │
├──────────────────────────────────────────┤
│  风格标签（如有）                         │
├──────────────────────────────────────────┤
│  底部 Footer:                             │
│  [中|EN] [自然语言|JSON]    [暂存到OMP]  │
└──────────────────────────────────────────┘
```

### 状态对应的右侧展示
| 选中任务状态 | 右侧内容 |
|-------------|---------|
| pending | "等待分析中..." 文字提示 |
| running | Loader2 + "正在分析图片..." |
| success | 完整内容（标题、提示词、分析、标签、按钮） |
| failed | 错误信息 + 重试按钮 + 移除按钮 |
| feedback | 成功反馈 + "关闭本次会话" 按钮 |

### Footer 布局
- 语言切换组（中/EN）
- 格式切换组（自然语言/JSON）
- 主操作按钮"暂存到OMP"

## Header 和整体布局设计

### 整体尺寸
- 弹窗宽度：600px（左侧 120px + 右侧 480px）
- 弹窗高度：700px
- 最大高度：700px（内容超出时右侧滚动）

### Header 结构
```
┌────────────────────────────────────────────────────┐
│  [Logo图标] Oh My Prompt          [最小化] [关闭] │
└────────────────────────────────────────────────────┘
```

### Header 行为
- Logo 图标：18x18，左上角
- 标题文字：`Oh My Prompt`，loading 状态显示 `分析中...`
- 最小化按钮：点击后弹窗缩小为 200x60，只显示 Header
- 关闭按钮：点击后关闭弹窗，清空所有任务

### 最小化状态
- 宽度：200px
- 高度：60px（只显示 Header）
- 标题显示统计：`3✓ / 1✗ / 2⟳`（成功/失败/进行中数量）
- 左侧列表和右侧内容都隐藏

### 拖拽行为
- Header 区域可拖拽移动弹窗位置
- 拖拽时 cursor 变为 `grabbing`
- 保持右侧边缘固定（最小化时位置调整逻辑与现有实现一致）

### ESC 关闭
- 按 ESC 键关闭弹窗，清空所有任务

## 实现要点

### 组件重构
1. VisionModal 新增 `selectedTaskId` state
2. VisionModal 新增左侧列表渲染逻辑
3. VisionModal 订阅 `useTaskQueueStore.tasks`
4. 移除 BatchProgressPanel 及相关引用

### 样式调整
1. 弹窗宽度从 480px 改为 600px
2. 新增左侧列表样式（120px 宽度、垂直布局、选中高亮）
3. 右侧保持现有样式，调整 padding 适配新宽度

### 状态同步
1. 任务添加时自动设置 `selectedTaskId` 为新任务
2. 任务状态变化时，如果当前选中的是 running → success，保持选中
3. 任务移除时，如果移除的是选中任务，切换到第一个任务或清空

## 测试要点

1. 单图添加 → 弹窗显示，左侧只有一个列表项
2. 多图添加 → 左侧显示多个列表项，可点击切换
3. 任务状态变化 → 状态图标正确更新
4. 最小化/展开 → 布局正确切换，位置保持一致
5. 拖拽移动 → 弹窗可拖拽，边缘固定
6. ESC 关闭 → 弹窗关闭，任务清空
7. 语言/格式切换 → 提示词内容正确切换
8. 复制/保存 → 功能正常工作
